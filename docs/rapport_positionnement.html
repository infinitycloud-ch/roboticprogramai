1. Architecture du Systeme

RoboticProgramAI est une architecture cognitive a 3 couches qui combine des agents LLM specialises, une simulation physique haute fidelite, et une memoire persistante avec distillation entre les runs.

┌─────────────────────────────────────────────────────────────────┐ │ COUCHE 1 — CERVEAU │ │ │ │ MonoCLI (Groq LLM) Scrib (Distillation) │ │ ├── Agent STRAT ──────── Knowledge Base persistante │ │ ├── Agent DEV Regles apprises entre runs │ │ └── Playbooks YAML Evolution cognitive mesurable │ │ │ ├─────────────────── CLI stdout (7 scripts) ──────────────────────┤ │ │ │ COUCHE 2 — INTERFACE CORPS │ │ │ │ mono_robot_sense.py mono_robot_look.py (camera) │ │ mono_robot_move.py mono_robot_goto.py (navigation) │ │ mono_robot_turn.py mono_robot_stand.py │ │ mono_robot_reset.py │ │ │ ├─────────────────── UDP bridge (25 Hz) ──────────────────────────┤ │ │ │ COUCHE 3 — MONDE │ │ │ │ Isaac Sim 5.1.0 + Isaac Lab 2.3.2 │ │ ├── Unitree Go2 (rough_model_7850.pt PPO) │ │ ├── Camera frontale 640x480 @ 2.5 Hz │ │ ├── Hospital Digital Twin (USD scene) │ │ └── DGX Spark (GB10, 128 GB, aarch64) │ │ │ └─────────────────────────────────────────────────────────────────┘

Principes cles

Separation perception/cognition — La couche Robotics fournit les outils de perception (camera, position, etat). La couche Cerveau raisonne, decide, et apprend. Aucune logique IA dans les scripts robot.

Interface CLI universelle — 7 scripts Python autonomes, zero dependance croisee, texte LLM-readable sur stdout. Tout agent LLM peut piloter le robot via SSH.

Memoire persistante — Les lecons apprises par le cerveau survivent entre les sessions. Le systeme Scrib distille les logs en regles actionnables stockees dans une Knowledge Base.

2. Resultats Locomotion

La politique de locomotion RL (PPO) a ete optimisee en passant du modele flat au modele rough, entrainee sur terrain accidente avec un reseau 7x plus large.

Metrique flat_model_6800 (ancien) rough_model_7850 (actuel) Amelioration
Reseau MLP 48 → 128³ → 12 235 → 512/256/128 → 12 7x parametres
Cycles avant chute 7 (puis fall) 17+ (aucune chute) +143%
Duree max continue ~50 secondes 160+ secondes +220%
Distance max parcourue 3.2 metres 22.8 metres (A/R) +623%
Stabilite Z moyenne 0.270 m 0.380 m +41%
Root cause identifiee : le feedback loop last_action dans le vecteur d'observation accumulait des erreurs apres 50s (2.5x la duree d'episode d'entrainement). Le modele rough, avec son reseau 4x plus large et son entrainement sur terrain difficile, est intrinsequement plus robuste a cette derive.

3. Evolution Cognitive — Sprint 16

Objectif : demontrer que le cerveau (LLM) apprend et s'ameliore entre les runs sans modification de code.

Protocole experimental

Phase A — 3 runs baseline : le robot navigue vers le docteur a position connue (5,0,0).

Phase B — Perturbation : le docteur est deplace a (5,3,0) sans informer le robot. Le cerveau doit percevoir et s'adapter.

Phase C — Evolution : meme config que Phase A, mais la KB contient les lecons de Phase B.

Differenciation unique

1. Architecture multi-agents hierarchique — STRAT planifie, DEV execute, Spark deploie. Chaque agent a un role defini et des outils specifiques. Aucun concurrent n'utilise cette approche.

2. Memoire persistante avec distillation — Les lecons apprises survivent entre les sessions. Le systeme Scrib transforme les logs bruts en regles actionnables. Les VLA concurrents encodent tout dans les poids du modele — aucune adaptation post-training.

3. Evolution cognitive quantifiee — Nous mesurons l'amelioration : -6.7% distance, +17.5% resilience. Aucun concurrent ne publie de metriques d'evolution cognitive equivalentes.

4. Plateforme accessible + vertical deploye — Le Go2 EDU est le quadrupede le plus accessible du marche. Notre digital twin hospitalier est un vertical pret a demontrer, pas un papier de recherche.
